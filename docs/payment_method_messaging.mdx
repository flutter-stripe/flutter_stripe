# Payment Method Messaging

### Promote Buy Now, Pay Later options to your customers.

> **Note:** The Payment Method Messaging element wraps Stripe's preview `PaymentMethodMessagingElement` API (`@_spi` on iOS, `@OptIn(PaymentMethodMessagingElementPreview::class)` on Android), so its visual output may change in future SDK releases. It is supported on **iOS and Android only**.

The `PaymentMethodMessaging` widget renders a native, auto-resizing view that tells customers how a purchase could be paid for using Buy Now, Pay Later (BNPL) payment methods. Stripe fetches and localizes the messaging for the given amount and currency, so no server-side endpoint is required.

## 1. Set up Stripe [Client Side]

First, you need a Stripe account. [Register now](https://dashboard.stripe.com/register).

The Flutter SDK is open source and fully documented. To install it:
 - Run the command `flutter pub add flutter_stripe`
 - This adds a line to your project's `pubspec.yaml` with the latest package version.

For details on the latest SDK release and past versions, see the [Releases](https://github.com/flutter-stripe/flutter_stripe/releases) page on GitHub.

When your app starts, configure the SDK with your Stripe [publishable key](https://dashboard.stripe.com/) so that the element can request messaging from the Stripe API.

```dart
void main() async {
  Stripe.publishableKey = stripePublishableKey;
  runApp(const App());
}
```

Use your [test mode](https://stripe.com/docs/keys#obtain-api-keys) keys while you develop, and your [live mode](https://stripe.com/docs/keys#test-live-modes) keys when you publish your app.

## 2. Add the widget [Client Side]

Add the `PaymentMethodMessaging` widget anywhere in your widget tree — for example near a product price or on your checkout screen. Provide a `PaymentMethodMessagingConfiguration` describing the purchase. The widget sizes itself to the rendered messaging; when there is nothing to show (for example the amount is below the minimum for the available payment methods) it collapses to zero height.

```dart
import 'package:flutter_stripe/flutter_stripe.dart';

PaymentMethodMessaging(
  configuration: PaymentMethodMessagingConfiguration(
    amount: 9999,
    currency: 'usd',
    countryCode: 'US',
    paymentMethods: [
      PaymentMethodMessagingPaymentMethod.klarna,
      PaymentMethodMessagingPaymentMethod.afterpayClearpay,
      PaymentMethodMessagingPaymentMethod.affirm,
    ],
  ),
);
```

The widget reacts to configuration changes: updating `amount`, `currency` or `paymentMethods` and rebuilding the widget refreshes the messaging in place.

## 3. Configuration parameters

`PaymentMethodMessagingConfiguration` accepts the following parameters:

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `amount` | `int` | Yes | The amount intended to be collected, in the smallest currency unit (for example `100` for $1.00 USD). |
| `currency` | `String` | Yes | The three-letter ISO currency code, in lowercase (for example `usd`). |
| `paymentMethods` | `List<PaymentMethodMessagingPaymentMethod>` | Yes | The payment methods to request messaging for. Values: `klarna`, `afterpayClearpay`, `affirm`. If empty, Stripe uses your dashboard preferences. |
| `countryCode` | `String?` | No | The two-letter country code of the customer's location. If omitted, the country is determined from the IP address. |
| `locale` | `String?` | No | The locale used to localize the messaging. Defaults to the device locale. See [supported locales](https://docs.stripe.com/js/appendix/supported_locales). Not all device locales are supported; English is used as a fallback. |

## 4. React to the rendered height (Optional)

The widget manages its own height, but you can observe height changes — for example to animate surrounding layout or hide an empty placeholder — via the `onHeightChange` callback. A reported height of `0` means there is no messaging to display for the current configuration.

```dart
PaymentMethodMessaging(
  configuration: configuration,
  onHeightChange: (height) {
    debugPrint('Messaging height: $height');
  },
);
```
